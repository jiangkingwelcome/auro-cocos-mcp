💪 我们的独占优势

安全 & 防护

  • 24 字节随机 Token 认证 (X-MCP-Token)
  • IP 级速率限制 (240/min 固定窗口)
  • 本地回环限制 (127.0.0.1 / ::1)
  • 请求超时保护 (20s)
  • 请求体大小限制 (1MB)
  • confirmDangerous 破坏性操作拦截
  • AI_RULES 强制行为约束

原子操作 & 智能

  • create_prefab_atomic (多阶段 + 自动回滚)
  • import_and_apply_texture (导入 + 应用一步到位)
  • setup_ui_layout (ScrollView 层级一键创建)
  • create_tween_animation_atomic (关键帧动画)
  • auto_fit_physics_collider (碰撞体自动适配)
  • batch 操作 ($N.uuid 引用，最大 200 步)
  • register_custom_macro (运行时注册自定义工具)

AI 智能辅助

  • 组件名自动纠正 ("sprite" → "Sprite")
  • 操作后自动高亮受影响节点
  • 错误提示附带组件名建议列表
  • 场景快照对比 (scene_snapshot + scene_diff)
  • 深度场景验证 + 修复建议
  • 资源验证 + 结构化修复建议
  • 节点包围盒查询 (2D/3D)
  • 按图层查找节点 (bitmask)

工程 & 生态

  • generate_script 脚本生成器
  • create_material 材质创建器
  • animation_tool (10 actions: 播放/暂停/crossfade/关键帧)
  • physics_tool (10 actions: 碰撞体/刚体/关节/物理世界)
  • slice_sprite 九宫格切图
  • batch_import 批量导入资源
  • 灯光系统 (3 接口: 查询/创建/属性设置)
  • 场景环境控制 (ambient/shadows/fog/skybox + 5 预设)
  • 材质高级操作 (Shader 宏/自定义材质/克隆/technique/灰度)
  • 坐标转换 (screen⇄world) + 摄像机截图
  • 9 种几何体 (box/sphere/cylinder/cone/plane/torus/capsule/quad)
  • 脚本绑定系统 (attach/detach + 编译检查 + @property 查询)
  • 批量属性设置 (set_component_properties)
  • 10 个单元测试文件 (Vitest)

  • 热更重启 (无需重启编辑器)
  • 工具管理系统 (启用/禁用)
  • 偏好设置三级 scope (global/project/default)

🎨 5.2 开发者体验 (DX)

当前不足

  • 面板 UI 简陋 — 纯 HTML/CSS，第一印象差
  • 无 i18n 多语言支持 — 仅中文界面
  • 安装后无引导流程 — 新用户无从下手
  • 错误信息仅中文 — 国际用户无法理解
  • 无交互式调试面板 — 无法在面板内测试工具

提升方案

  • Vue 3 面板重写 — 实时状态、工具列表、日志查看
  • i18n 引擎 — 至少中/英双语，语言自动检测
  • 首次使用引导 — 向导式配置 + 一键验证连接
  • 双语错误信息 — errorCode + 中英文 message
  • 面板内 Playground — 可直接输入 action 测试

增强方向描述预期价值难度

MCP Prompts 模板预置常见工作流 Prompt 模板：「帮我搭建 UI」「优化场景性能」「创建角色控制器」降低 AI 使用门槛，标准化输出质量低

🔧 5.4 工程质量与稳定性

可改进项

  • 速率限制使用固定窗口 — 突发流量处理不够平滑
  • 无集成测试 — 仅单元测试，缺少端到端验证
  • 无自动化 CI/CD — 发布流程手动
  • 场景脚本热更新困难 — 需重启编辑器
  • 日志系统不够结构化 — 排查问题依赖人工
  • db:// 路径规范化未实现 — 接口已预留

提升方案

  • 滑动窗口限速 — 更平滑的流量控制
  • E2E 测试套件 — Playwright 驱动编辑器自动化测试
  • GitHub Actions CI — 自动测试 + 版本发布
  • 场景脚本热重载 — 监听文件变更，自动 re-inject
  • 结构化日志 (JSON) — 支持 severity / traceId / timestamp
  • 路径规范化落地 — db:// ↔ 绝对路径 ↔ 相对路径 互转

🔌 5.6 MCP 协议演进 — 前瞻性布局

MCP 标准新能力

  • Resources — 静态数据暴露（场景树 / 资源清单 / 项目配置）
  • Prompts — 预设工作流模板（一键创建 UI 页面）
  • Sampling — 服务端主动调用 LLM（自动生成代码）
  • Roots — 工作区文件系统访问
  • Logging — 结构化日志流

适配计划

  • Resources 暴露: scene://tree, asset://list, project://info
  • Prompts 模板: 「搭建登录页」「优化渲染性能」「添加角色控制」
  • Sampling 应用: 场景分析 → 自动生成优化建议脚本
  • Roots 集成: 直接读写项目 assets/ 目录
  • Logging 输出: 操作日志 + 性能指标实时流

📋 对比总结

我们的优势：安全机制（5 层防护 vs 无防护）、原子操作（5 个事务性宏 vs 无）、AI 行为护栏（AI_RULES + confirmDangerous）、 单元测试覆盖（10 个文件 vs 1 个），独立动画工具（animation_tool, 10 actions）、独立物理工具（physics_tool, 10 actions）， 完整灯光系统（3 接口覆盖 DirectionalLight / SpotLight / SphereLight），场景环境控制（ambient / shadows / fog / skybox + 5 种一键预设）， 材质高级操作（Shader 宏 / 项目自定义材质分配 / 克隆 / technique 切换 / Sprite 灰度化）， 坐标转换（screen⇄world）、摄像机离屏截图和脚本绑定系统（attach_script 一键挂载 + 批量属性 + 编译状态检查 + @property 查询）， 以及 9 种几何体、批量操作、九宫格切图、批量导入和自定义宏注册等独占功能。

竞品的优势：市场先发（539 stars + Cocos Store 在售），已建立的用户基础和社区认知度。 竞品本地工具虽无显式认证，但本地回环天然提供了一定程度的隔离。

持平领域：构建系统、渲染统计、参考图片管理、偏好设置、工具管理、事件广播等核心编辑器操作能力基本对等。 功能广度上我们已全面超越（19 工具 / 236 actions vs 50 工具 / ~150 操作码），尤其在灯光、环境、材质高级操作领域形成了显著差距。

待改进：速率限制使用固定窗口而非滑动窗口，db:// 路径规范化功能预留了接口但尚未实现。 下一步应聚焦于 Cocos 商城上架和社区推广。
