1. 执行摘要

支持引擎（Cocos/Godot/Unity）

CocosDesignTool 是一款面向游戏开发者的 AI 原生工具集，包含可视化蓝图编辑器（GameFlowEditor）和多引擎 MCP 桥接插件。产品通过 Model Context Protocol 让 AI 编码助手（Claude、Cursor、Windsurf 等）直接操控游戏项目，实现"对话式游戏开发"。


当前市场处于 AI + 游戏开发工具的早期爆发期，竞品数量有限但增长迅速。我们的核心差异化在于：

  • 漏斗架构：19 工具 vs 竞品 50-80 个扁平工具，Token 开销降低 60%+
  • 多引擎覆盖：唯一同时支持 Cocos + Godot + Unity 的 MCP 方案
  • 原子操作 + 自动回滚：竞品不具备的企业级可靠性
  • 向下兼容：支持 Cocos 3.4.2+（竞品要求 3.8.0+）

核心结论：建议采用 开源核心 + 付费增值（Pro 版） 的混合模式，在 6 个平台同步上架，预计 12 个月内可实现 ¥15-40 万 年收入（保守-乐观区间）。

2. 产品概览与核心优势

GameFlowEditor — 可视化蓝图编辑器

  • 拖拽式节点编辑（START/DIALOGUE/CHOICE/EVENT/END）
  • 实时对话预览与流程验证
  • 自动导出为游戏引擎脚本
  • 多关卡/场景管理
  • MCP Server 模式，AI 可直接编辑蓝图
  • 撤销/重做完整操作历史
  • 中英文双语 UI

Cocos MCP Bridge — AI 桥接插件（最成熟）

  • 19 工具 / 230+ 动作，漏斗架构
  • 场景操作 ~65 动作 / 场景查询 ~40 动作
  • 资源管理 ~31 动作 / 编辑器控制 ~42 动作
  • 原子宏工具（create_prefab_atomic 等）
  • 组件名自动规范化 / 路径规范化
  • 危险操作拦截 / 错误自愈建议
  • 一键注入 11 款 AI IDE 配置
  • 可配置安全策略（速率限制/回环限制/自动回滚）

3.1 Cocos Creator 生态

分析：Cocos 生态竞争者极少，daxianlee 是唯一有影响力的开源竞品。我们在工具数量（230+ vs 80）、架构设计（漏斗 vs 扁平）、版本兼容性（3.4.2 vs 3.8.0）上全面领先。关键差距：社区影响力（566 星 vs 我们较少）。

5.1 各平台上架详细要求

GitHub 优化清单

  • 完善 README：添加 GIF 演示、架构图、快速开始
  • 添加 GitHub Topics：mcp, cocos-creator, godot, unity, ai-tools, game-dev
  • 创建 Release 页面，附带预编译包
  • 添加 GitHub Actions CI/CD 徽章
  • 创建 CONTRIBUTING.md 引导社区贡献
  • 设置 GitHub Discussions 社区讨论
  • 添加 GitHub Sponsors 赞助入口
  • 提交到 awesome-mcp、awesome-cocos 等列表

npm 发布清单

  • 包名建议：@cocosdesigntool/mcp-bridge
  • 确保 package.json 的 description、keywords 完善
  • 添加 bin 入口支持 npx cocos-mcp-bridge
  • 编写 npm README（与 GitHub 可不同）
  • 设置 .npmignore 排除测试/文档
  • 发布 scoped package 避免命名冲突

Cocos Store 上架要求

  • 需要 Cocos 开发者账号
  • 插件需符合 Cocos 插件规范
  • 提供至少 3 张截图 + 1 个演示视频
  • 详细的中英文描述
  • 标注支持的 Cocos Creator 版本范围
  • 审核周期约 3-7 个工作日
  • 可选免费或付费（30% 抽成）

Product Hunt 发布准备

  • 提前 2 周准备 Maker 页面
  • 制作 1 分钟产品演示视频
  • 准备 5 张高质量产品截图
  • 撰写简洁有力的 Tagline（≤60 字符）
  • 建议 Tagline："AI-native game dev toolkit for Cocos, Godot & Unity"
  • 选择周二/周三发布（流量最高）
  • 提前联系 5-10 个 Hunter 帮忙推荐
  • 发布当天在社交媒体同步宣传
