# MCP 完整人工测试协议

本协议约定：**回归与验收时，能力验证必须经 MCP 桥接完成**，并在 `tests/interactive-test-plan.html` 中留下可追溯记录。其目的是模拟真实 AI/自动化调用路径，避免「只在编辑器里手点、从未走 `tools/call`」造成的漏测。

---

## 1. 什么是「经 MCP 测试」

满足以下任一方式即视为**经 MCP**：

1. **HTTP JSON-RPC**：向编辑器内插件提供的 `POST http://127.0.0.1:<端口>/mcp` 发送 JSON-RPC，使用 `tools/call` 调用 `tests/test-cases.json` 中对应 tool/action 与参数（请求头携带 `X-MCP-Token`）。
2. **集成脚本**：在 Cocos 已启动、插件已加载的前提下，运行 `node tests/integration/mcp-full-test.mjs`（内部同样走 `/mcp`），其结果可作为 AI 基线，**不能替代**你对每条用例的人工结论，但可加速冒烟。

**不算经 MCP（禁止作为「已测」依据）**：

- 仅在 Cocos 菜单里操作、从未发 `tools/call`；
- 仅阅读文档或静态检查源码，未对应用例请求体实际调用；
- 仅运行 Vitest 单元测试而不覆盖该条 MCP 场景（除非用例明确标注为纯单元项）。

---

## 2. 完整测试流程（推荐顺序）

1. **环境**：打开目标 Cocos 版本与测试工程，启用 Aura 插件，确认 HTTP 服务与 Token（面板或 `.mcp-token` / `connection-info`）。
2. **自动化基线（可选但推荐）**：执行 `npx vitest run`；在编辑器就绪后执行 `node tests/integration/mcp-full-test.mjs`，更新或对照 `tests/test-report.json` / 页面中的 AI 基线。
3. **逐条 MCP 人工执行**：打开 `tests/interactive-test-plan.html`（需已生成 `interactive-test-plan.data.js`），按 Tool/阶段筛选，**对每条用例**：
   - 若用例在 `tests/test-cases.json` 中带有 **前置条件** 与 **建议前置步骤（MCP）**，须**先按顺序完成前置**（将占位符如 `<canvas-uuid>` 换为实际 uuid），再执行主 **「请求输入」**；不得在未满足前置时仅对主调用打「通过」。
   - **`needsRetest`**：若用例标记为需复测（此前在部分环境、替代对象或未完成前置时记过「通过」），须在**完整前置**下重新执行 MCP 验证；复测通过后可在 **人工备注** 写明「已按前置复测」并视情况将 `manualVerification` 在仓库中更新为最终结论。
   - 用 MCP 客户端或自写脚本发送与用例 **「请求输入」** 一致的 `tools/call`；
   - 对照 **预期结果** 与编辑器实际表现（含场景树、资源、控制台）；
   - 在页面中更新 **人工测试结果** 与备注。
4. **问题与存疑必须落库**：
   - **确认缺陷**：人工结果选「失败」或「阻塞」，在 **问题描述** 中写现象、复现步骤、是否阻塞发布；
   - **拿捏不准**：选 **「存疑」**，在 **人工备注** 或 **问题描述** 中写明不确定点、待谁确认、建议复现环境；
   - **轻微备注**：即使选「通过」，若有关键注意事项，可写在 **人工备注**。
5. **收尾**：勾选页面左侧 **执行检查**；填写测试信息、发布结论；**保存本地 JSON** 或导出 CSV，纳入版本附件或 CI 产物。

---

## 3. 与双轨测试台字段的对应关系

| 行为 | 页面操作 |
|------|----------|
| MCP 调用结果与预期一致 | 人工测试结果 → **通过** |
| 明确失败 | → **失败** + **问题描述** |
| 环境/前置不满足 | → **阻塞** + 备注原因 |
| 结果依赖环境、需求澄清或需二次确认 | → **存疑** + **人工备注** / **问题描述** |
| 缺陷等级 | **严重级别** |
| 跟踪单 | **问题单 / 链接** |

---

## 4. 「完整」的判定标准（建议）

- **用例维度**：计划内每条用例均有非「待测」的人工结论（通过 / 失败 / 阻塞 / 存疑），且存疑项在发布前有责任人或复测安排。
- **MCP 维度**：P0/P1 用例均已实际发送过与用例一致的 `tools/call`（或由 `mcp-full-test.mjs` 覆盖且你已人工确认无偏差）。
- **记录维度**：`interactive-test-plan` 导出 JSON 已保存，问题与存疑可追溯。
