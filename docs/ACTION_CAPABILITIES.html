scene_query 场景只读查询 42 actions ▶
场景结构
tree 查询 获取整个场景的节点树形结构，就像编辑器层级面板看到的那样。可选 includeInternal 查看引擎内部隐藏节点。
list 查询 获取场景中所有节点的扁平列表（uuid、名称、深度、子节点数）。快速浏览场景全部节点。
stats 查询 获取场景统计概览：节点总数、激活节点数、场景名称等。适合快速了解场景规模。
node_detail 查询 查看某个节点的完整详情：位置、旋转、缩放、所有组件列表、是否激活、Layer 等。
find_by_path 查询 通过层级路径查找节点，例如 "Canvas/Panel/Button"。快速定位已知路径的节点。
节点信息
get_components 查询 列出某节点上挂载的所有组件（Sprite、Label、自定义脚本等）。
get_parent 查询 获取某节点的父节点信息。
get_children 查询 获取某节点的直接子节点列表。
get_sibling 查询 获取某节点的同级兄弟节点列表。
get_world_position 查询 获取节点的世界坐标位置 {x, y, z}。
get_world_rotation 查询 获取节点的世界旋转角度。
get_world_scale 查询 获取节点的世界缩放值。
get_active_in_hierarchy 查询 检查节点在层级中是否真正激活（考虑父节点链的 active 状态）。
get_node_bounds 查询 获取节点的边界框。2D 节点返回 UITransform 的 local/world rect；3D 节点返回 AABB 包围盒。
搜索与筛选
find_nodes_by_name 查询 按名称子串搜索场景中的节点。例如搜索 "Button" 找到所有包含该名称的节点。
find_nodes_by_component 查询 找出场景中所有挂载了某个组件的节点，如找出所有有 Sprite 的节点。
find_nodes_by_layer 查询 按 Layer 掩码筛选节点。例如找出所有 UI_2D 层节点。
组件属性读取
get_component_property 查询 读取节点上某个组件的某个属性值，例如读取 Label 的 string、Sprite 的 spriteFrame。
get_node_components_properties 查询 一次性获取节点上所有组件的所有属性，完整的组件属性快照。
摄像机 & Canvas
get_camera_info 查询 获取摄像机详细参数：FOV、近远裁剪面、投影类型、清除标志、优先级、视口矩形等。不传 uuid 则查所有摄像机。
get_canvas_info 查询 获取 Canvas 组件信息（设计分辨率、适配模式等）。
场景全局
get_scene_globals 查询 获取场景全局配置（ambient、fog、shadows 等原始数据）。
get_scene_environment 查询 获取结构化的场景环境设置：环境光（天空颜色、亮度）、阴影、雾效、天空盒、八叉树，分子系统返回。
灯光 & 材质
get_light_info 查询 获取场景中所有灯光组件信息：类型、颜色、亮度、范围、阴影设置、位置旋转。支持按 uuid 过滤。
get_material_info 查询 获取节点渲染器上的材质信息：Effect 名称、Technique、Pass 列表、Uniform 值（mainColor、roughness、metallic 等）。
动画 & 碰撞器
get_animation_state 查询 获取 Animation 组件状态：动画片段列表、是否播放中、当前时间、默认 Clip。
get_collider_info 查询 获取节点上所有碰撞器组件的详情（形状、大小、偏移）以及 RigidBody 信息。
坐标转换
screen_to_world 查询 将屏幕坐标通过摄像机转换为世界坐标。可指定摄像机 uuid，否则自动找。
world_to_screen 查询 将世界坐标通过摄像机转换为屏幕坐标。判断物体在屏幕上的位置。
脚本查询
check_script_ready 查询 检查一个脚本类是否已编译并注册到引擎中。脚本刚创建后需等待编译，用此接口轮询就绪状态。
get_script_properties 查询 获取脚本类的所有 @property 声明：属性名、类型、默认值、是否可见。用于了解脚本可设置哪些属性。
编辑器选择 & 焦点
get_current_selection 查询 获取编辑器中当前选中的节点及其详细信息。
get_active_scene_focus 查询 获取 AI 上下文焦点：优先返回选中节点详情，无选中时返回场景统计作为兜底。
list_all_scenes 查询 列出项目中所有 .scene 文件的 db:// 路径和 UUID。
验证 & 诊断
validate_scene 查询 对当前场景进行验证检查：空节点、缺失组件、重复同名兄弟、过深嵌套等。
deep_validate_scene 查询 深度验证场景：检测缺失资源引用、孤立节点，并提供修复建议和质量评分。
detect_2d_3d 查询 检测当前场景是 2D、3D 还是混合模式，帮助 AI 选择合适的组件和操作。
performance_audit 查询 对场景进行性能审计：节点过多、层级过深、Draw Call 估算等，并给出优化建议。
list_available_components 查询 列出引擎中所有可用的组件类型（含自定义脚本），按类别分组显示。
快照 & 导出
scene_snapshot 查询 捕获当前场景的完整状态快照，用于稍后与另一个快照做对比（scene_diff）。
scene_diff 查询 比较两次快照，找出新增、删除和修改的节点。追踪操作前后的场景变化。
export_scene_json 查询 将整个场景节点树导出为 JSON 格式数据。
measure_distance 查询 测量两个节点之间的 2D/3D 距离（世界坐标）。

scene_operation 场景写操作 64 actions ▶
节点增删改
create_node操作 在场景中新建一个空节点，可指定父节点和插入位置。
destroy_node危险 永久删除一个节点及其子节点（需 confirmDangerous=true 确认）。
reparent操作 将节点移到另一个父节点下，改变层级结构。
duplicate_node操作 克隆一个节点（可选是否包含子节点），返回新节点的 UUID。
clear_children危险 删除某节点的全部子节点（需 confirmDangerous=true 确认）。
group_nodes操作 创建一个空父节点，将多个节点编组到它下面。
变换 (Transform)
set_position操作 设置节点的本地坐标位置 (x, y, z)。
set_rotation操作 设置节点的本地欧拉旋转角度 (度)。
set_scale操作 设置节点的本地缩放值（1.0 = 100%）。
set_world_position操作 设置节点的世界坐标位置。
set_world_rotation操作 设置节点的世界旋转角度。
set_world_scale操作 设置节点的世界缩放值。
reset_transform操作 重置节点变换：位置归零、旋转归零、缩放回 1。可选择性重置。
节点属性
set_name操作 重命名节点。
set_active操作 启用或禁用节点（active = true/false）。
set_layer操作 设置节点的 Layer 层掩码（如 DEFAULT、UI_2D）。
set_anchor_point操作 直接设置 UITransform 锚点 (anchorX, anchorY)。
set_content_size操作 直接设置 UITransform 内容尺寸 (width, height)。
排序 & 编辑器状态
move_node_up操作 在兄弟列表中将节点上移一位。
move_node_down操作 在兄弟列表中将节点下移一位。
set_sibling_index操作 精确设置节点在兄弟中的排序位置（0 开始）。
lock_node操作 在编辑器层级面板中锁定节点，防止误操作。
unlock_node操作 解锁被锁定的节点。
hide_node操作 在编辑器中隐藏节点（不影响运行时 active）。
unhide_node操作 取消隐藏节点。
组件操作
add_component操作 给节点添加一个组件（如 Sprite、Label、RigidBody2D 或自定义脚本）。
remove_component操作 从节点上移除一个组件。
set_property操作 设置节点上某个组件的某个属性值，如 Label.string = "Hello"。
reset_property操作 将组件属性重置为默认值。
reset_node_properties操作 将节点上所有（或指定）组件属性重置为默认值。
call_component_method操作 调用组件上的方法，如 Animation.play()、AudioSource.stop()。
batch_set_property操作 对多个节点批量设置同一组件的同一属性值。
脚本绑定
attach_script操作 一键挂载脚本到节点：自动 addComponent + 批量设置 @property 值 + 重复检测。原子操作。
set_component_properties操作 一次调用批量设置组件的多个属性键值对，替代多次 set_property。
detach_script操作 从节点上移除一个自定义脚本组件。
对齐 & 剪贴板
align_nodes操作 多节点对齐：左对齐、右对齐、水平居中、顶对齐、底对齐、垂直居中、均匀分布。
clipboard_copy操作 复制节点到编辑器剪贴板。
clipboard_paste操作 从剪贴板粘贴节点到指定父节点下。
预制体
create_prefab操作 将节点保存为 .prefab 预制体资源文件。
instantiate_prefab操作 将预制体实例化到场景中。
enter_prefab_edit操作 进入预制体编辑模式。
exit_prefab_edit操作 退出预制体编辑模式。
apply_prefab操作 将预制体实例的更改应用回预制体资源。
revert_prefab操作 撤销预制体实例的修改，恢复为原始预制体状态。
validate_prefab操作 检查预制体文件的完整性和依赖。
UI 组件创建
create_ui_widget操作 快速创建完整 UI 组件：Button、Label、Sprite、Toggle、Slider、ProgressBar、EditBox。
setup_particle操作 创建粒子系统节点，支持预设模板：火焰、烟雾、雨、雪、火花、爆炸。
create_skeleton_node操作 创建骨骼动画节点（Spine 或 DragonBones）。
generate_tilemap操作 创建 TiledMap 瓦片地图节点。
create_primitive操作 创建 3D 几何体：Box、Sphere、Cylinder、Cone、Plane、Torus、Capsule、Quad，支持颜色和阴影。
摄像机
create_camera操作 创建新摄像机节点，可设置 FOV、近远平面、投影类型、位置。
set_camera_property操作 批量修改摄像机属性：FOV、裁剪面、投影、清除标志、背景色、视口矩形、光圈快门 ISO 等。
set_camera_look_at操作 让摄像机朝向一个世界坐标点。
camera_screenshot操作 通过摄像机进行离屏渲染截图（RenderTexture）。
材质
set_material_property操作 设置渲染器材质的 Uniform 属性，如 mainColor、roughness、metallic。
assign_builtin_material操作 给渲染器分配内置材质（builtin-standard、builtin-unlit、builtin-2d-sprite），可设颜色。
assign_project_material操作 将项目中的自定义 .mtl 材质资源分配给渲染器。
set_material_define操作 设置材质的 Shader 编译宏（如 USE_ALBEDO_MAP）并重编译。
clone_material操作 克隆共享材质为独立实例，安全修改不影响其他对象。
swap_technique操作 切换材质的渲染 Technique 索引。
sprite_grayscale操作 一键将 Sprite 切换为灰度模式或恢复彩色。
灯光
create_light操作 创建灯光节点：平行光、聚光灯、球形点光源，设置颜色/亮度/范围/阴影/色温。
set_light_property操作 修改灯光属性：亮度、颜色、范围、锥角、阴影设置（PCF / Bias / 距离）、色温等。
场景环境
set_scene_environment操作 修改场景环境：ambient（天空颜色/亮度）、shadows（阴影）、fog（雾效）、skybox（天空盒），或直接用 5 种一键预设（白天/日落/夜晚/室内/大雾）。
事件绑定
bind_event操作 绑定 UI 事件回调：click、toggle、slider、editbox、scrollview、pageview 等 9 种事件类型。
unbind_event操作 解绑 UI 事件：按 handler 名或索引移除，或清空全部。
list_events操作 列出节点上所有已绑定的 UI 事件回调。
音频 & 物理
audio_setup操作 给节点添加 AudioSource 组件并配置音量、循环、自动播放。
setup_physics_world操作 配置物理世界参数：重力方向/大小、休眠、时间步长。
批量 & 流水线
batch操作 批量执行多个操作序列，支持 $N.uuid 引用前一步结果。最多 200 步。
